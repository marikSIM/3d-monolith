# 🔧 ГИБРИДНАЯ AI СИСТЕМА 3D MONOLITH

## 📋 Обзор

Гибридная AI система предоставляет **три режима работы** с возможностью переключения прямо во время использования:

| Режим | Иконка | Описание | Требования |
|-------|--------|----------|------------|
| **📚 База знаний** | 📚 | Быстрые ответы из проверенной базы | Работает всегда |
| **🤖 Локальная LLM** | 🤖 | Полноценная языковая модель | GGUF модель (2-8 ГБ) |
| **🌐 WebLLM** | 🌐 | Браузерная модель через WebGPU | WebGPU в системе |

---

## 🚀 Быстрый старт

### 1. Запуск приложения

```bash
npm start
```

### 2. Открытие AI чата

Нажмите на кнопку **🔧 Fixik AI** в правом нижнем углу экрана.

### 3. Переключение режимов

Внизу окна чата находится переключатель режимов:

```
┌─────────────────────────────────┐
│  📚    🤖    🌐                │
│  База  LLM   Web               │
├─────────────────────────────────┤
│ ✓ Режим готов                   │
│ База знаний — быстро и надёжно  │
└─────────────────────────────────┘
```

## 📚 Режим 1: База знаний (Knowledge Base)

**Режим по умолчанию.** Работает сразу после запуска.

### Преимущества:
- ✅ Мгновенные ответы
- ✅ Не требует дополнительного ПО
- ✅ Работает на любом железе
- ✅ Проверенная информация

### Что умеет:
- 🖨️ Настройка принтеров (176+ моделей)
- 📦 Параметры материалов (8+ типов)
- 🔧 Диагностика неисправностей (6+ категорий)
- ⚙️ Расчёт параметров печати
- 🛡️ Рекомендации по безопасности

### Примеры запросов:
```
• "Как настроить Ender 3?"
• "Какая температура для PLA?"
• "Отслоение углов, что делать?"
• "Чем отличается ABS от PETG?"
```

## 🤖 Режим 2: Локальная LLM

Полноценная языковая модель, работающая **локально на вашем устройстве**.

### Требования:
- **Модель:** GGUF формат (TinyLlama, Llama, Mistral и др.)
- **Размер:** 0.5–8 ГБ (рекомендуется 2-4 ГБ)
- **RAM:** 8+ ГБ рекомендуется
- **CPU:** AVX2 поддержка

### Как загрузить модель:

1. **Скачайте GGUF модель:**
   - [TinyLlama-1.1B-Chat-v1.0-GGUF](https://huggingface.co/TheBloke/TinyLlama-1.1B-Chat-v1.0-GGUF)
   - [Llama-2-7B-Chat-GGUF](https://huggingface.co/TheBloke/Llama-2-7B-Chat-GGUF)
   - [Mistral-7B-Instruct-GGUF](https://huggingface.co/TheBloke/Mistral-7B-Instruct-v0.2-GGUF)

2. **В AI чате:**
   - Нажмите на кнопку **🤖 LLM**
   - Если модель не загружена, появится предложение загрузить
   - Выберите `.gguf` файл через диалог

3. **После загрузки:**
   - Модель автоматически активируется
   - Путь сохраняется в настройках Electron
   - Автоматическая загрузка при следующем запуске

### Преимущества:
- ✅ Умные ответы на любые вопросы
- ✅ Понимание контекста разговора
- ✅ Работа без интернета
- ✅ Конфиденциальность (всё локально)

### Недостатки:
- ⏱️ Первый ответ может занять 5-30 секунд
- 💾 Требует 2-8 ГБ места на диске
- 🖥️ Нагрузка на CPU/RAM

### Примеры запросов:
```
• "Объясни как работает FDM печать"
• "Напиши руководство по калибровке стола"
• "Какие есть способы улучшить адгезию?"
• "Сравни все популярные типы филаментов"
```

## 🌐 Режим 3: WebLLM

Браузерная языковая модель с использованием **WebGPU**.

### Требования:
- **WebGPU:** Поддержка в браузере/Electron
- **GPU:** Видеокарта с поддержкой WebGPU
- **Драйверы:** Актуальные драйверы

### Проверка WebGPU:

```javascript
// В консоли браузера
if (navigator.gpu) {
    console.log('✅ WebGPU доступен');
} else {
    console.log('❌ WebGPU недоступен');
}
```

### Преимущества:
- ✅ Не требует загрузки модели
- ✅ Использует GPU ускорение
- ✅ Современная архитектура

### Недостатки:
- ❌ Требуется поддержка WebGPU
- ❌ Может быть недоступно на старых системах
- ❌ Зависит от интернета (загрузка модели)

## 🔄 Переключение режимов

### Через UI:
1. Откройте AI чат
2. Нажмите на кнопку нужного режима
3. Система проверит доступность
4. Режим переключится (или покажет ошибку)

### Через консоль:
```javascript
// Переключение на базу знаний
await window.hybridAI.switchMode('knowledge-base');

// Переключение на локальную LLM
await window.hybridAI.switchMode('local-llm');

// Переключение на WebLLM
await window.hybridAI.switchMode('webllm');

// Проверка доступных режимов
window.hybridAI.getAvailableModes();

// Получение текущего режима
console.log(window.hybridAI.currentMode);
```

## 🛠️ Технические детали

### Архитектура:

```
┌─────────────────────────────────────────────┐
│           Renderer Process                  │
│  ┌─────────────────────────────────────┐    │
│  │     HybridAISystem (hybrid-ai.js)   │    │
│  │  ┌──────────┐ ┌──────────┐ ┌──────┐│    │
│  │  │Knowledge │ │ Local    │ │ Web  ││    │
│  │  │Base      │ │ LLM      │ │ LLM  ││    │
│  │  └──────────┘ └──────────┘ └──────┘│    │
│  └─────────────────────────────────────┘    │
└─────────────────────────────────────────────┘
                    ↕ IPC
┌─────────────────────────────────────────────┐
│            Main Process (Electron)          │
│  ┌─────────────────────────────────────┐    │
│  │  electron-store (настройки)         │    │
│  │  node-llama-cpp (интеграция)        │    │
│  │  knowledge-base.json (база)         │    │
│  └─────────────────────────────────────┘    │
└─────────────────────────────────────────────┘
```

### Хранение данных:

**Настройки AI:**
- Путь к модели: `store.get('modelPath')`
- Текущий режим: `store.get('aiMode')`
- Настройки: `store.get('aiSettings')`

**База знаний:**
- Файл: `data/knowledge-base.json`
- Загрузка через IPC: `load-knowledge-base`

**Пользовательские данные:**
- Путь: `app.getPath('userData')`
- Electron Store автоматически

### IPC методы:

```javascript
// Загрузка модели
window.electron.loadAIModel(modelPath)

// Проверка сохранённой модели
window.electron.getSavedAIModel()

// Генерация ответа
window.electron.generateAIResponse(prompt, options)

// Получение режима
window.electron.getAIMode()

// Установка режима
window.electron.setAIMode(mode)

// Загрузка базы знаний
window.electron.loadKnowledgeBase()

// Проверка WebGPU
window.electron.checkWebGPU()
```

## 🔍 Диагностика

### Режим не переключается:

**Проблема:** Кнопка режима неактивна (серая)

**Решение:**
1. Проверьте доступность режима:
   ```javascript
   window.hybridAI.getAvailableModes()
   ```
2. Для LLM: загрузите модель
3. Для WebLLM: проверьте WebGPU

### Модель не загружается:

**Проблема:** Ошибка при загрузке GGUF

**Решение:**
1. Проверьте размер файла (0.5-8 ГБ)
2. Убедитесь что файл `.gguf`
3. Проверьте целостность файла

### База знаний не работает:

**Проблема:** Ответы не находятся

**Решение:**
1. Проверьте наличие `data/knowledge-base.json`
2. Проверьте консоль на ошибки
3. Перезагрузите приложение

- ✅ Нужен быстрый ответ

- ✅ Нужны точные параметры материала

## 🔐 Безопасность

### Система безопасности включает:

1. **Валидация запросов** — блокировка опасных команд
2. **Валидация ответов** — проверка перед показом
3. **Предупреждения** — уведомления о рисках
4. **Блокировка** — полная блокировка опасных советов

### Красные линии (блокируются всегда):
- ❌ Отключение питания при нагретых компонентах
- ❌ Разборка хотэнда без остывания
- ❌ Работа с фотополимерами без СИЗ
- ❌ Превышение температурных лимитов
