# ⚙️ УСТАНОВКА И НАСТРОЙКА 3D MONOLITH AI

## 📋 Системные требования

### Минимальные:
- **ОС:** Windows 10 / macOS 10.15 / Linux
- **Процессор:** Intel Core i5 / AMD Ryzen 5
- **ОЗУ:** 8 ГБ
- **Место на диске:** 500 МБ (без учёта моделей AI)
- **Node.js:** v18+

### Рекомендуемые:
- **ОС:** Windows 11 / macOS 12+ / Ubuntu 22.04+
- **Процессор:** Intel Core i7 / AMD Ryzen 7
- **ОЗУ:** 16 ГБ (для локальной LLM)
- **Видеокарта:** С поддержкой WebGPU (для WebLLM режима)
- **Место на диске:** 10 ГБ (с моделями AI)

---

## 🚀 Быстрая установка

### Шаг 1: Установка Node.js

1. Скачайте Node.js LTS: https://nodejs.org/
2. Установите (галочка "Automatically install necessary tools" для Windows)
3. Проверьте установку:
   ```bash
   node --version
   npm --version
   ```

### Шаг 2: Установка зависимостей

```bash
cd "3D MONOLITH"
npm install
```

### Шаг 3: Запуск приложения

```bash
npm start
```

## 📦 Установка зависимостей AI

### Автоматическая установка (рекомендуется)

При первом запуске приложение автоматически установит:
- ✅ `electron-store` — хранение настроек
- ✅ `node-llama-cpp` — локальная LLM

Если автоматическая установка не сработала:

```bash

```

## 🤖 Настройка AI режимов

### Режим 1: База знаний (по умолчанию)

**Ничего настраивать не нужно!**

Просто запустите приложение и откройте AI чат (🔧).

### Режим 2: Локальная LLM

#### 1. Скачайте GGUF модель

**Рекомендуемые модели:**

| Модель | Размер | Качество | Ссылка |
|--------|--------|----------|--------|
| **TinyLlama-1.1B** | ~2 ГБ | Базовое | [Скачать](https://huggingface.co/TheBloke/TinyLlama-1.1B-Chat-v1.0-GGUF/resolve/main/tinyllama-1.1b-chat-v1.0.Q4_K_M.gguf) |
| **Mistral-7B** | ~4 ГБ | Хорошее | [Скачать](https://huggingface.co/TheBloke/Mistral-7B-Instruct-v0.2-GGUF/resolve/main/mistral-7b-instruct-v0.2.Q4_K_M.gguf) |
| **Llama-2-7B** | ~4 ГБ | Хорошее | [Скачать](https://huggingface.co/TheBloke/Llama-2-7B-Chat-GGUF/resolve/main/llama-2-7b-chat.Q4_K_M.gguf) |

#### 2. Загрузите модель в приложение

1. Откройте AI чат (нажмите 🔧)
2. Нажмите кнопку **🤖 LLM**
3. Выберите скачанный `.gguf` файл
4. Дождитесь загрузки (5-30 секунд)

#### 3. Используйте

Модель автоматически активируется и будет загружаться при каждом запуске.

### Режим 3: WebLLM

**Требования:**
- Видеокарта с поддержкой WebGPU
- Актуальные драйверы

#### Проверка WebGPU:

Откройте консоль (F12) и введите:
```javascript
if (navigator.gpu) {
    console.log('✅ WebGPU доступен');
} else {
    console.log('❌ WebGPU недоступен');
}
```

Если WebGPU доступен — кнопка **🌐 Web** будет активна.

## 🐛 Решение проблем

### Ошибка: "require() of ES Module ... not supported"

**Решение:** Обновите `electron-store`:
```bash
npm install electron-store@latest --save
```

### Ошибка: "node-llama-cpp не инициализирован"

**Причина:** Модуль не установлен или не совместим

**Решение:**
```bash
npm rebuild node-llama-cpp
```

Или используйте режим базы знаний (📚).

### Ошибка: "WebGPU недоступен"

**Причина:** Видеокарта не поддерживает WebGPU или устарели драйверы

**Решение:**
1. Обновите драйверы видеокарты
2. Используйте режим базы знаний или локальную LLM

### Приложение не запускается

**Решение:**
1. Удалите `node_modules` и `package-lock.json`
2. Выполните `npm install`
3. Запустите `npm start`

### Модель долго загружается

**Причина:** Большой размер модели или медленный диск

**Решение:**
1. Используйте модели меньше (1-2 ГБ)
2. Используйте режим базы знаний

## 📊 Проверка установки

### В приложении:

1. Откройте консоль (F12)
2. Введите:
   ```javascript
   // Проверка гибридной AI системы
   window.hybridAI.getStats()
   
   // Проверка доступных режимов
   window.hybridAI.getAvailableModes()
   ```

### Ожидаемый результат:

```javascript
{
  mode: 'knowledge-base',
  webgpuAvailable: false,  // или true если есть WebGPU
  modelLoaded: false,      // или true если загружена модель
  knowledgeBaseLoaded: true,
  isInitialized: true
}
```
